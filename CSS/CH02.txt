# 4. :ROOT SELECTOR

```css

  font-size: 1em;
```

* Used to define:

  * Global font-size
  * CSS variables

# 6. EMS (em)

## Definition:

* `1em = current elementÔÇÖs font-size`

---

## Behavior:

### Non-font properties:

* Based on **current element**

### font-size:

* Based on **inherited font-size**

---

## Example:

```css
padding: 1em;
```

---

# 7. EM GOTCHA

```css

  font-size: 1.2em;
  padding: 1.2em;
}
```

* font-size Ôćĺ parent-based
* padding Ôćĺ computed font-size

Ôćĺ Same value, different result

# 10. REM (ROOT EM)

## Definition:

* `1rem = root font-size`

---

## Example:

```css
:root {
  font-size: 1em;
}

p {
  font-size: 1rem;
}
```

---

## Advantage:

* Stable
* No nesting issues


## Why use `em` at root?

* Respects user settings
* Keeps scaling flexible

## Key Insight:

## New Units:

* Dynamic Ôćĺ `dvh`, `dvw`

## Recommended:

# 17. RESPONSIVE FONT (ADVANCED)

## Using calc():

```css
font-size: calc(0.5em + 1svw);
```

---

## Using clamp() (BEST):

```css
font-size: clamp(0.9rem, 0.6rem + 0.5svw, 1.5rem);
```

---

## Why this works well:

* Root uses `em`
* clamp uses `rem + vw`
  Ôćĺ Fully scalable system

---

Two other related functions that may be useful from time to time are
min() and max(). Min resolves to the smallest of the given values (e.g., width:
min(200px, 20svw);). Max resolves to the largest of the given values (e.g.,
min-height: max(200px, 20svw);)
